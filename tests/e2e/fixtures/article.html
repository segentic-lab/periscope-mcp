Understanding the Cascade

The CSS cascade is the algorithm that decides which declaration wins when several rules target the same element. It weighs origin, specificity, and source order, in that priority, to resolve conflicts deterministically.

Specificity

Specificity is a three-part score computed from the selectors in a rule. Inline styles outrank IDs, IDs outrank classes and attributes, and those in turn outrank type selectors. When two declarations tie, the later one wins.

For a deeper reference, see the MDN specificity guide, which documents the exact weighting rules and worked examples.

Example

a.button { color: blue; }
#cta { color: red; }   /* wins: IDs outrank classes */
  • Origin and importance are considered first.
  • Then specificity of the matching selector.
  • Finally, document source order breaks remaining ties.
